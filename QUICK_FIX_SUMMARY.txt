ENUM WIDGET FIX - QUICK SUMMARY
================================================================================

ISSUE:
  - level field (and other enum fields) was showing as array widget
  - Had +/- buttons instead of dropdown
  - Schema defines: "type": "string" with "enum": [...]
  - Should render as ComboBox dropdown, not ArrayWidget

ROOT CAUSE:
  - widget_factory.cpp getFieldType() was checking for enum before type
  - Returned FieldType::Array for all enums
  - createWidget() then used new ArrayWidget() instead of ComboBox

FIX APPLIED:
  - Moved enum check to createWidget() BEFORE field type determination
  - If enum exists → call createEnumWidget() directly → creates QComboBox
  - If no enum → proceed with normal type checking

FILES CHANGED:
  - src/ui/widget_factory.cpp (2 methods modified)
